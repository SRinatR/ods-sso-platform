# Инструкция администратора партнёра ODS

ODS разделяет три разных адреса:

- `https://auth.ods.uz` — только регистрация, вход, consent и OAuth/OIDC endpoints.
- `https://partners.ods.uz` — общий вход в партнёрский кабинет: список ваших компаний и создание
  новых компаний.
- `https://{slug}.ods.uz` — рабочий кабинет конкретной компании: SSO-приложения, участники,
  роли, branding consent-экрана, аудит и аналитика.

## 1. Создание компании

1. Зарегистрируйте личную учётную запись на `https://auth.ods.uz/register`.
2. Подтвердите email и войдите.
3. Откройте `https://partners.ods.uz`.
4. Укажите название, сайт, юридическое название и контактный email.
5. Проверьте предложенный код компании. Для Tatarlar используется `tatarlar`.
6. После создания откроется изолированный кабинет `https://tatarlar.ods.uz`.

Один человек может создать несколько компаний. Они отображаются на
`https://partners.ods.uz`, но настройки каждой компании открываются только на её поддомене.

Контактный email — это адрес для уведомлений и связи. Он не выдаёт доступ автоматически. Доступ
выдаётся только через раздел «Участники и роли».

## 2. Подключение приложения

Для Tatarlar создайте confidential OIDC client:

- название: `Tatarlar production`
- callback URL: `https://api.tatarlar.uz/api/v1/auth/sso/callback`
- post logout URL: `https://tatarlar.uz/`
- scopes: `openid profile email`
- client type: `confidential`
- token endpoint auth: `client_secret_basic`

PKCE S256 обязателен автоматически.

После создания:

1. Скопируйте `client_id`.
2. Скопируйте `client_secret` и сохраните в менеджере секретов. Повторно он не показывается.
3. Внесите значения в настройки Tatarlar через его административный интерфейс или передайте
   ответственному за production env Tatarlar по защищённому каналу.
4. Не отправляйте secret в чат, задачу, скриншот или обычное письмо.

## 3. Участники

Пользователь сначала регистрирует обычную учётную запись ODS. Затем владелец компании добавляет
его email в разделе «Участники и роли»:

- Редактор (`editor`) — работа с содержимым;
- Пользователь (`user`) — обычная работа в сервисе;
- Читатель (`viewer`/`reader`) — только чтение.

Владелец (`owner`) один и не может быть понижен через кабинет.

Если пользователь открывает `https://tatarlar.ods.uz`, но видит сообщение «Нет доступа к
компании», значит его ODS-аккаунт ещё не добавлен в участники Tatarlar. Владелец или администратор
должен добавить email этого пользователя в разделе «Участники и роли».

## 4. Проверка входа

1. Откройте кнопку «Войти с помощью SSO ODS» в Tatarlar.
2. Браузер должен перейти на `https://auth.ods.uz/authorize`.
3. После входа и согласия браузер вернётся на callback Tatarlar.
4. Проверьте, что роль в Tatarlar соответствует роли в кабинете ODS.
5. Отключите тестового участника в ODS и убедитесь, что новый вход ему запрещён.

При обращении в поддержку передавайте время ошибки, URL и `x-request-id`, но не cookies,
authorization code, access token, refresh token или client secret.

## 5. Аналитика

В кабинете `https://tatarlar.ods.uz` владелец или администратор видит блок «Аналитика».
Он показывает агрегированные данные за последние 30 дней:

- успешные SSO-входы;
- уникальных пользователей;
- refresh token операции;
- выдачу и отзыв consent;
- изменения приложений, участников и secret;
- security-события, например повторное использование refresh token.

Аналитика scoped только на текущую компанию и её OIDC-приложения. Кабинет не показывает
пароли, токены, client secret, сырые IP-адреса или полный user-agent.
